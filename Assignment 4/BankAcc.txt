**Bank Account Management System Assignment**

**Objective:**
Develop a bank account management system in Java that allows users to create accounts, perform transactions, and view account details. The system should include features such as creating different types of accounts, handling transactions, and generating account statements.

**Features to Implement:**

1. **Account Classes:**
   - Create classes for different types of bank accounts (e.g., savings account, checking account). Each class should include attributes such as account number, balance, and account holder information.

2. **Transaction Handling:**
   - Implement methods to handle transactions, including deposits and withdrawals. Ensure proper validation for withdrawal transactions to avoid overdrawing.

3. **Account Management:**
   - Develop a class (e.g., `Bank`) to manage the bank accounts. Include methods for:
     - Creating a new account.
     - Closing an account.
     - Transferring funds between accounts.

4. **User Interface:**
   - Create a simple user interface to interact with the bank accounts. Use the console for input and output. Allow the user to choose different operations.

5. **Data Validation:**
   - Implement validation checks for user input, such as ensuring a valid account number is entered during transactions.

6. **Interest Calculation:**
   - Implement a mechanism to calculate and apply interest to savings accounts periodically.

**Submission Guidelines:**

1. **Code Submission:**
   - Submit the complete Java source code for your bank account management system.

2. **Documentation:**
   - Provide clear documentation explaining the design choices, structure of classes, and any external libraries or APIs utilized (if any).

3. **User Guide:**
   - Include a user guide on how to use the bank account management system. Explain the available operations, input formats, and any error handling.

4. **Testing:**
   - Test your program thoroughly, especially considering edge cases like creating accounts, transferring funds, and handling transactions with insufficient balance.

5. **Reflection:**
   - Write a reflection on the challenges faced, lessons learned, and potential improvements for future iterations of the bank account management system.

**Tips:**
- Break down the problem into smaller tasks before starting the implementation.
- Make use of Java's Object-Oriented Programming principles to design classes and manage bank accounts efficiently.
- Consider implementing data structures like ArrayList or HashMap to store and manipulate account records.
